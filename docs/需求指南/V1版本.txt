用户提的需求：
-------------------
1.我们跟银行合作，会在管理平台给合作商户配置补贴额度，需要一个AI客服，在代理商和银行的群里，可以根据提供的商户号，由AI客服实现，查询系统的活动使用额度并答复咨询人。
2.我们每个月要给银行提供报表和对账单，模版都是固定的，发送时间也是固定的。数据来源是从内部管理平台下载的N多账单文件，根据模版字段，进行加工整理后发给银行对账。
-------------------

整理的设计文档（我需要设计一个通用的 excel 解析读取，以及通用的 excel 根据匹配规则自动生成）：
-------------------
大概开发两个主页面的工具应用
第一个页面：配置页面，1.全部配置显示 2.通过excel上传配置 3.配置可编辑


第二个页面：通用excel数据读取页面


其他功能页面自行帮我设计


1.上传解析配置表（保存明细数据表的解析过程），保存excel怎么解析excel数据的配置，根据哪一行获取标题行，从哪行开始是数据行，解析截止到哪列，这个功能我希望用户随便上传一个数据excel后，然后显示加载基础的数据，让用户可以在前端进行框选，通过框选保存解析配置到后端数据库表。每次上传excel文档，可以指定对应的数据读取解析配置来设置当前excel的数据到底如何解析的。解析明细数据导入到明细表

2.上传解析模板配置表（保存模板配置数据的解析过程），解析excel模板中保存的多条配置信息，模板配置信息里面存的是数据处理相关的规则，比如根据某列按月展示等等的一些多维表格的配置规则。解析规则数据导入到规则表

3.根据用户选择的规则，对明细数据进行处理和预览，然后可以让用户导出当前预览的excel数据出来

要求：
前端：使用React，在项目根目录的 frontend目录下开发
后端：使用 python，在项目根目录的 backend目录下开发
使用 uv 管理器管理依赖
新增依赖：uv add xxx依赖包
服务启动：uv run main.py

数据库使用 sqlite 本地数据库，保存到当前项目后端根目录下 db 目录
日志按yyyyMMdd.log 的格式按天保存到当前项目后端根目录下 log 目录
